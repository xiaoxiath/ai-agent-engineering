# 贡献指南

感谢你对《AI Agent 工程》的关注！这是一个开源图书项目，我们欢迎各种形式的贡献。

## 贡献方式

### 1. 报告问题

- 技术错误、过时信息、链接失效等，请提交 Issue
- 请尽量描述清楚问题所在的章节和位置

### 2. 内容贡献

- **修正补充**：修正错误、补充遗漏的技术细节
- **代码示例**：增加可运行的代码示例
- **案例研究**：分享你的 Agent 开发实战经验
- **翻译**：将内容翻译为其他语言

### 3. 贡献流程

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/improve-chapter-5`)
3. 提交你的修改 (`git commit -m 'Add detailed compaction implementation example'`)
4. 推送到分支 (`git push origin feature/improve-chapter-5`)
5. 创建 Pull Request

## 写作规范

### 文件命名

- 章节文件：`ch{序号}-{英文短名}.md`
- 代码示例：放在 `code-examples/ch{序号}-{主题}/` 目录下

### 内容格式

- 使用 GitHub Flavored Markdown
- 一级标题用于章节名，二级标题用于小节
- 代码块标明语言（typescript, yaml, bash 等）
- 专业术语首次出现时给出中英文对照

### 代码示例

- 主要语言：TypeScript
- 代码必须可以独立运行或清楚标明伪代码
- 添加必要的注释说明
- 遵循项目的 ESLint 配置

## 沟通渠道

- **GitHub Issues**：功能建议、Bug 报告
- **GitHub Discussions**：技术讨论、内容规划
- **Pull Requests**：代码和内容贡献

## 行为准则

- 尊重每一位贡献者
- 建设性地讨论技术问题
- 保持开放和包容的态度

感谢你的贡献！🙏
