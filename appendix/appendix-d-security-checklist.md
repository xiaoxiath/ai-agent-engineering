# 附录 D：Agent 安全检查清单

## D.1 设计阶段安全审查

### D.1.1 威胁建模

- [ ] 已识别所有数据流和信任边界
- [ ] 已列举 OWASP ASI01-ASI10 中适用的威胁
- [ ] 已为每个威胁制定缓解措施
- [ ] 已定义安全等级和对应策略

### D.1.2 架构安全

- [ ] Agent 系统 prompt 与用户输入严格隔离
- [ ] 工具调用实现了最小权限原则
- [ ] 敏感操作需要人工审批 (HITL)
- [ ] 沙箱环境隔离了代码执行
- [ ] 超时和资源限制已配置

## D.2 开发阶段安全检查

### D.2.1 Prompt Injection 防护

- [ ] 输入清洗层 (InputSanitizer) 已实现
- [ ] 系统 prompt 包含注入防御指令
- [ ] 输出验证器检查未授权指令执行
- [ ] 间接注入（来自工具返回）已防护
- [ ] 已进行红队测试验证防护效果

### D.2.2 数据安全

- [ ] 传输层加密 (TLS 1.3)
- [ ] 静态数据加密
- [ ] PII 数据自动检测和脱敏
- [ ] API Key 和凭据使用密钥管理服务
- [ ] 日志中不包含敏感信息

### D.2.3 工具安全

- [ ] 工具参数严格验证（类型、范围、格式）
- [ ] 文件操作限制在允许的路径内
- [ ] HTTP 请求白名单限制域名
- [ ] 数据库查询仅允许只读操作（分析场景）
- [ ] 工具调用频率限制已配置

## D.3 部署阶段安全检查

### D.3.1 基础设施

- [ ] 容器运行在非 root 用户下
- [ ] 网络策略限制了出入站流量
- [ ] 密钥管理使用 Vault 或云 KMS
- [ ] 健康检查和存活探针已配置
- [ ] 自动扩缩容策略防止资源耗尽

### D.3.2 监控与告警

- [ ] 异常行为检测规则已配置
- [ ] Prompt injection 尝试的告警
- [ ] API 用量异常的告警
- [ ] 敏感操作审计日志开启
- [ ] 事故响应流程已制定并演练

## D.4 运营阶段安全检查

### D.4.1 持续安全

- [ ] 定期红队测试（至少每季度一次）
- [ ] 依赖库漏洞扫描自动化
- [ ] LLM 模型更新后回归测试
- [ ] 安全事件复盘和修复流程
- [ ] 用户反馈的安全问题响应 SLA

### D.4.2 应急预案

| 场景 | 应急措施 | 恢复时间 |
|------|---------|---------|
| Prompt 注入攻击 | 阻断恶意会话 + 规则更新 | < 1h |
| 数据泄露 | 密钥轮转 + 影响评估 | < 4h |
| 模型幻觉导致错误操作 | 回滚操作 + 人工接管 | < 30min |
| DDoS/API 滥用 | 限流 + IP 封禁 | < 15min |
| 第三方服务故障 | 降级到本地模型/缓存 | < 5min |

## D.5 安全评分卡

```
总分 = Σ(已检查项 / 总检查项) × 100

90-100: 优秀 ✅ - 可上线
75-89:  良好 ⚠️ - 修复高风险项后上线
60-74:  及格 🟡 - 需要重点改进
<60:    不合格 ❌ - 需要重新设计安全架构
```
