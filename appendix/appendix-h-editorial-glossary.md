# 附录 H：术语与结构说明

本附录用于统一全书中最核心的结构术语，降低章节间重复定义和概念漂移。

## 1. 结构层级

### 1.1 主干章节

指构成 Agent 工程主线的章节，默认优先级为 **P0**。这些章节回答的是“如何构建一个可靠的 Agent 系统”。

### 1.2 扩展章节

指与生产化、安全、观测、部署、多 Agent 等直接相关的章节，默认优先级为 **P1**。这些章节回答的是“如何让 Agent 系统在复杂场景下可控、可评估、可运维”。

### 1.3 主题选读章节

指协议、体验、案例等特定主题内容，默认优先级为 **P2**。

### 1.4 参考与前沿章节

指附录、索引、高时效性资料、趋势判断等，默认优先级为 **P3**。

## 2. 核心术语

### 2.1 Agent Loop

指 Agent 在“感知—推理—行动—反馈”之间循环的核心执行机制。

### 2.2 Context Engineering

指围绕“在正确时间把正确信息放进上下文窗口”而展开的系统工程，而不仅仅是 Prompt 编写。

### 2.3 Tool

指 Agent 可直接调用的最小外部能力单元，通常具有明确的输入、输出、参数约束和执行边界。

### 2.4 Skill

指建立在 Tool 之上的高层复合能力，通常封装了一组步骤、约束或工作流。Skill 可以由一个或多个 Tool 组成。

### 2.5 Memory

指 Agent 为跨轮次、跨任务或跨会话保留信息而构建的记忆系统。它与上下文窗口不同，也不等同于 RAG 知识库。

### 2.6 RAG

指检索增强生成。其核心是从外部知识源检索相关信息，并将其注入到生成阶段，以提升事实性和覆盖面。

### 2.7 Eval

指用于衡量 Agent 系统行为质量、稳定性、成本和安全性的评估活动与评估系统。

### 2.8 AX

指 Agent Experience，即面向 Agent 系统的人机交互体验设计，关注控制感、透明度、可恢复性与信任校准。

## 3. 正文与代码的关系

本书正文的首要目标是解释知识结构和工程方法，代码示例的首要目标是支撑关键概念落地。因此：

- 正文不追求展示所有复杂实现细节
- 代码目录不承诺所有章节均提供完整项目
- 阅读时请结合 `CODE_EXAMPLE_MATRIX.md` 判断当前示例成熟度
